Proxy

Extends \Roulette\Base

Proxy class is used for database operation, it operate SELECT, INSERT, UPDATE, DELETE.

Any class can use proxy to make operation to database then proxy forward process onto \Roulette\Roulette\Adapter. One proxy proxy for one table, to operate CRUD in another table use better to use another table. Any custom operation can made with method query.

 // dont forget to set the adapter first, only once
 // eg, use CI3 framework
 Proxy::tunnel(new \Roulette\proxies\CI3\Proxy($this->db));

Proxy can be used for CRUD in SQL notation (SELECT, INSERT, UPDATE, DELETE)

 // INSERT operation using proxy
 Proxy::insert('employees', array(
     'name'=>'me', 'job'=>'programmer'
 ));

 // UPDATE operation using proxy
 Proxy::update('employees', array(
     'name'=>'me' // where name is `me`
 ), array(
     'job'=>'developer' // set job = `developer`
 ));

 // SELECT operation using proxy
 Proxy::select('employees', array(
     'limit'=> 3,
     'filters' => array( 'job'=>'developer' )
 )); // will return array of record from database

 // DELETE operation using proxy
 Proxy::delete('employees', array(
     'name'=>'me' // where name is `me`
 ));

Operate select to database.

select(Array $config = null, String|Array $callback = null) : Mixed
static

Example:

 // #using fields
 // fields can be writen in string or array (numeric or associative)
 Proxy::select(array(
     'table' => 'users',
    'fields' => array('name','email') // it can be writen as string as well, eg: 'name, email'
 ));
 // if writen in string
 Proxy::select(array(
     'table' => 'users',
    'fields' => 'name,email'
 ));
 // or using array as alias
 Proxy::select(array(
     'table' => 'users',
    'fields' => array('name'=>'Name') // will same as 'select `name` as `Name`'
 ));

 // #using filters
 // filters is like fields, could be writen in string or array
 Proxy::select(array(
     'table' => 'users',
     'filters' => 'name = `me`'
 ));
 // or
 Proxy::select(array(
     'table' => 'users',
     'filters' => array('name'=>'me')
 ));
 // and easy to use multi filters
 // if you are familiar with CI where condition style, it will be easy
 Proxy::select(array(
     'table' => 'users'
     'filters' => array('email like'=>'%@gmail.com', 'ocupation IS NOT NULL'=>null, 'age >'=>5)
 ));

 // #using sorter
 // sorter can be writen in string or array too
 Proxy::select(array(
     'table' => 'users'
     'sorter' => 'name' // will be sort by name asc
 ));
 // or
 Proxy::select(array(
     'table' => 'users'
     'sorter' => 'name desc' // will be sort by name desc
 ));
 // or
 Proxy::select(array(
     'table' => 'users'
     'sorter' => array('name') // will be sort by name asc
 ));
 // or
 Proxy::select(array(
     'table' => 'users'
     'sorter' => array('name'=>'desc') // will be sort by name desc
 ));
 // or use multiple sort
 Proxy::select(array(
     'table' => 'users'
     'sorter' => array('name', 'age'=>'desc') // will be sort by name asc and sort by age desc
 ));

 // #using limmit and sort
 Proxy::select(array(
     'table' => 'users'
     'limit' => 10 // default is null, mean no limit
 ));
 // or
 Proxy::select(array(
     'table' => 'users'
     'limit' => 10,
     'start' => 5  // default is 0
 ));

 // #using result_array
 Proxy::select(array(
     'table' => 'users'
     'result_array' => false // each record will be return as object instead of array, default result_array is true
 ));

Arguments

$config

Array

Select config. Config can be contain: fields@Array|String, filters@Array|String, sorter@Array|String, limit@Integer, start@Integer, result_array@Boolean

$callback

Response
